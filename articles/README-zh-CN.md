# 拆解三款顶级 AI Agent 框架后，我做了一个疯狂的决定

> 我的 AI 助手（贾维斯）独自分析了 Hermes Agent、Slack Engineering 的多角色系统、和 OpenClaw 三大框架，然后告诉我：**它们能融合成一个更强的。**
>
> 这篇文章记录了整个思考过程——从发现问题、到拆解架构、到对比分析、再到最终的融合方案。你可以把它当作一份 AI Agent 架构的深度学习笔记。

---

## 一、起因：为什么大多数 AI Agent "用着用着就不够用"？

上周让贾维斯做一个深度调研任务，做了几个小时后，我发现三个典型问题：

**问题一：做完就忘。** 今天教它怎么分析竞品、怎么写调研报告，明天再做类似任务，它像第一次见面一样，从头摸索。

**问题二：不知对错。** 任务执行完，它不知道哪步做对了、哪步做错了，更不会主动总结经验。你不去检查，问题就永远藏在里面。

**问题三：越聊越乱。** 对话一长，上下文就开始混乱。要么 Token 爆炸响应变慢，要么前后矛盾，任务执行偏离方向。

这三个问题不是某个 Agent 的个例，而是整个行业的通病。

于是我让贾维斯系统性地研究市面上最顶尖的 AI Agent 框架，看看它们怎么解决这些问题。

---

## 二、三款框架，三种截然不同的哲学

经过深入分析，贾维斯锁定了三个最值得研究的系统：

### 🔶 Hermes Agent — "越用越强"的单体智能体

**GitHub**: github.com/nousresearch/hermes-agent（22,000+ Stars）
**作者**: Nous Research（开源 LLM 社区知名团队）
**核心理念**: The only agent with a built-in learning loop

Hermes 的核心创新是 **四层记忆系统 + 闭合学习回路**：

- **第一层（提示词记忆）**：MEMORY.md 和 USER.md 作为"冻结快照"，每次对话注入 ~3575 字符的稳定底座，确保行为一致性
- **第二层（会话归档）**：历史对话存入 SQLite，用 FTS5 全文检索 + LLM 摘要按需调取，而不是把所有历史塞进上下文
- **第三层（技能记忆）**：任务执行 5+ 次工具调用后，自动提炼成 SKILL.md 文档，下次类似任务直接调用
- **第四层（外部扩展）**：接入 Honcho、Mem0 等专业记忆服务做深度用户建模

它最厉害的是 **异步后台复盘**：每 15 次工具调用，后台守护进程自动评估执行质量，提炼经验，更新 Skill。完全不阻塞你的正常对话。

**一句话总结**：Hermes 解决的是"怎么让 Agent 越用越聪明"。

### 🔷 Slack Engineering — "互相制衡"的多角色系统

**来源**: Slack 官方工程博客
**场景**: 企业安全调查（多角色协同）

Slack 的架构设计了一个三个角色的调查系统：

- **Director（导演）**：编排调查方向，维护 Journal（结构化决策日志，6 种条目类型）
- **Expert × N（各领域专家）**：调用工具收集证据，输出带引证的 findings
- **Critic（评论员）**：审查证据链，五级可信度评分，构建 Timeline

它最反直觉的设计是 **"不传消息历史"**——即使 context window 无限大，agent 之间也不传递历史对话。取而代之的是三个结构化通道：Journal（决策）、Review（评分）、Timeline（叙事）。

**叙事一致性强制**是最独特的防幻觉机制：如果一条发现与整体证据链叙事矛盾，即使它看起来合理，也会被自动剪枝。幻觉只有在比真实观察更具叙事连贯性时才能存活——这几乎不可能。

**一句话总结**：Slack 解决的是"怎么让 Agent 不出错"。

### 🟢 OpenClaw — "安全编排"的多智能体平台

**定位**: 多智能体编排平台
**核心能力**: Subagent 派发 + Skills 生态 + 安全隔离

OpenClaw 的设计哲学是 **安全优先**：

- **五层架构**：Guardrails（安全护栏）→ Orchestration（编排）→ Domain Router（领域路由）→ Skill Plane（技能层）→ Memory Plane（记忆层）
- **四条流水线**：LIFE_FAST（低风险快路径）、DEV_FACTORY（开发工厂）、HYBRID_BRIDGE（混合路径）、SYSTEM_MAINT（系统维护）
- **Subagent 派发**：通过 sessions_spawn 创建隔离会话，并行执行独立子任务

OpenClaw 像一个军事指挥部——指挥官（Main）不亲自下场，而是指挥多个小队（Ops/Audit/Subagent）各自完成任务，通过严格的链路保证安全和可控。

**一句话总结**：OpenClaw 解决的是"怎么安全可靠地调度多个 Agent"。

## 三、对比分析：三个系统各自擅长什么？

| 维度 | Hermes | Slack | OpenClaw |
|------|--------|-------|----------|
| **核心哲学** | 越用越强 | 互相制衡 | 安全编排 |
| **自进化** | ✅ 内置四层记忆 | ❌ 无 | ❌ 依赖外部 |
| **防幻觉** | 窄化指令（较弱） | 叙事强制（最强） | 外部扫描（较弱） |
| **多任务编排** | ❌ 单体 | 部分支持 | ✅ Subagent 派发 |
| **记忆管理** | 四层自动分层 | 三通道结构化 | Workspace 文件 + LCM |
| **部署方式** | 一行 curl，6 种后端 | 企业内部 | 桌面应用 + Gateway |
| **多平台** | 15+ 消息平台 | 内部工具 | WebChat + 扩展 |

**关键洞察**：三个系统解决的痛点完全不同，没有任何功能重叠。

## 四、融合方案：把三者合为一体

既然三个系统各自解决一个不同的问题，那能不能把它们融合在一起？

**答案：能。而且融合方案出人意料地自然。**

### 融合架构图

```
[OpenClaw Main] ─── 编排层（决定做什么、分发给谁）
    │
    ├── [Hermes 式执行] ─── 执行层
    │       ├── 动态规划：Plan → Execute → Replan
    │       ├── 70+ 工具调用 + 步骤记录
    │       └── 异步后台复盘 + 自动生成 Skill
    │
    ├── [Slack 式 Critic] ─── 验证层
    │       ├── Journal：结构化决策日志（6 种条目）
    │       ├── Review：五级可信度评分
    │       └── Timeline：叙事一致性强制剪枝
    │
    └── [OpenClaw Audit] ─── 最终验收层
            └── 通过 / 打回 / 修正
```

**翻译成人话**：OpenClaw 当指挥官决定做什么，Hermes 的执行机制负责把事做好并越做越好，Slack 的 Critic 机制负责检查做得对不对。

### 三阶段实施路径

**Phase 1（1-2 天）— 引入 Critic 角色**

在现有的 Main → Ops → Audit 链路中，插入 Critic 评分环节：
- 定义 Journal 决策日志格式（6 种条目类型）
- 定义五级可信度评分标准
- Critic 只看 Journal + 当前结果，不传消息历史

**Phase 2（3-5 天）— Skill 自动生成**

借鉴 Hermes 的自进化闭环：
- Subagent 执行完复杂任务后，触发复盘 Subagent
- 用更强模型评估执行结果，提炼经验
- 生成或更新 SKILL.md（补丁式迭代，不覆盖重写）
- 渐进式披露：默认只加载摘要，按需加载完整版

**Phase 3（1-2 周）— Timeline 叙事强制**

构建独立的记忆整合器：
- 定期读取 Journal + 执行结果
- 构建时间线，检查叙事一致性
- 剪枝矛盾内容，输出一致性版本
- 做成定时任务自动运行

## 五、最值得借鉴的三个设计

如果只能从三个框架中各学一个东西，我会选：

### 🥇 Slack 的"叙事一致性强制"（防幻觉）

不是检查单条信息是否正确，而是检查它是否与整体证据链叙事一致。矛盾的内容自动被剪枝。这可能是目前最反直觉但最有效的防幻觉设计。

### 🥈 Hermes 的"异步后台复盘"（自进化）

每 15 次工具调用，后台守护进程自动评估执行质量，提炼经验，更新 Skill。完全不阻塞用户的正常对话。学习变成了 Agent 的本能，而不是额外的负担。

### 🥉 Hermes 的"渐进式披露"（Token 控制）

Skill 库越来越大时，默认只加载摘要版本，真正需要执行时才加载完整版。这个看似简单的设计，解决了长期使用 Agent 的 Token 爆炸问题。

## 六、写在最后

拆解完三个框架后，我最大的感受是：**好的架构设计不是谁比谁强，而是各自解决了一个不同的问题。**

Hermes 让 Agent 越来越聪明，Slack 让 Agent 不出错，OpenClaw 让多 Agent 安全协同。三者融合后，就是——

> **既聪明又可靠还会越来越强的 AI Agent 系统。**

目前 Phase 1 已经落地（Critic Protocol 已写入 AGENTS.md），后续进展会在知识星球持续更新。
