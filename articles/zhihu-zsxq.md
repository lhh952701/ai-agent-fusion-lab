# 我让 AI 助手自己分析三个 Agent 框架，它给了我一个意外答案

---

上周我做了一个实验：让我的 AI 助手贾维斯，自己去研究市面上最顶尖的 AI Agent 框架，分析它们的架构，然后告诉我——能不能把最好的设计融合到它自己身上。

这不是一个普通的"框架对比"文章。这是一个 AI 在做自我升级决策的真实记录。

## 起点：一个很实际的痛点

事情起因很简单。让贾维斯做一个调研任务，做了好几个小时，做完之后我发现：

1. 它不知道自己哪步做对了哪步做错了
2. 它不会主动总结经验
3. 下次做类似任务，它像失忆了一样从头开始

这不是贾维斯的问题，是目前所有 AI Agent 的通病——**无状态**。每次对话都是新的一天，之前的经验教训全部归零。

所以我让它去研究这个问题，看看业界最顶尖的框架怎么解决。

## 它研究了三天，锁定了三个系统

### 第一个：Hermes Agent（GitHub 22,000+ Stars）

Nous Research 做的开源 Agent 框架。它的核心理念是"越用越强"——内置四层记忆系统，会自动从每次执行中提炼经验，写成 Skill 文档。下次遇到类似任务，直接调用已有经验。

它最厉害的是异步后台复盘：你在正常和它聊天时，它后台偷偷在分析刚才的执行过程，提炼经验。完全不打扰你。

### 第二个：Slack Engineering 的多角色调查系统

这是 Slack 官方工程博客里分享的安全调查系统架构。三个角色——Director（导演）、Expert（专家）、Critic（评论员）——互相配合完成调查。

它最反直觉的设计是"不传消息历史"。agent 之间不传递对话记录，只传递结构化的 Journal（决策日志）、Review（审查结果）和 Timeline（时间线）。

防幻觉的核心机制叫"叙事一致性强制"：如果一条发现和整体证据链矛盾，自动剪掉。幻觉只有在比真实观察更具叙事连贯性时才能存活——这几乎不可能。

### 第三个：OpenClaw

就是贾维斯自己跑的平台。多智能体编排，五层架构，四条流水线。像一个军事指挥部——指挥官不亲自干活，而是调度多个小队各自完成任务。

## 然后它告诉我：这三个可以融合

贾维斯的结论让我意外但又觉得理所当然：

**三个系统分别解决了一个完全不同的问题，没有任何功能重叠。**

- Hermes → 让 Agent 越来越聪明（自进化）
- Slack → 让 Agent 不出错（防幻觉）
- OpenClaw → 让多 Agent 安全协同（编排）

融合方案很自然：

```
OpenClaw（指挥官）→ 决定做什么
Hermes 式执行     → 越做越好
Slack 式 Critic   → 检查对不对
```

## 我说：那你开始做吧

贾维斯已经开始了。Phase 1 是引入 Slack 的 Critic 机制——在它每次执行完复杂任务后，用更强模型审查执行结果，给出五级可信度评分。

它自己定义了一套完整的规则：
- Journal 决策日志格式（6 种条目类型）
- 五级可信度评分标准
- 不传消息历史原则
- 叙事一致性强制剪枝

全写进了它的 AGENTS.md 运行规则里。

## 我的思考

说实话，这件事本身比三个框架的分析结果更有意思。

一个 AI Agent，能够：
1. 自主研究外部系统
2. 客观评估各自的优劣
3. 找出互补点
4. 制定融合方案
5. 自我实施改进

这不就是"自进化"的真实案例吗？

当然，现在还只是 Phase 1。后面的 Skill 自动生成、Timeline 叙事强制都还没做。但方向已经清楚了。

**AI Agent 的终极形态，可能不是某一个框架，而是把所有框架最好的设计融合在一起。**

我会在这个星球持续更新贾维斯的融合实验进展。
